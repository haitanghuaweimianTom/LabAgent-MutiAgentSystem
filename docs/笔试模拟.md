### 1. 请解释 RAG（Retrieval-Augmented Generation）的工作原理，并说明它解决了 LLM 的什么问题。

**参考答案**：

```
RAG = 检索增强生成

工作原理：
1. 检索阶段：用户查询 → 从知识库中检索相关文档片段
2. 增强阶段：将检索到的文档作为上下文，与用户查询一起输入 LLM
3. 生成阶段：LLM 基于增强后的上下文生成答案

解决的问题：
1. 知识时效性：LLM 训练数据有截止日期，RAG 可以检索最新信息
2. 领域知识：LLM 对专业领域知识有限，RAG 可以注入领域文档
3. 减少幻觉：LLM 可以基于检索到的事实生成答案，而不是编造
4. 可追溯性：答案可以引用检索到的文档来源

典型架构：
- 索引：文档分块 → embedding → 存入向量数据库
- 检索：查询 embedding → 向量相似度搜索 → Top-K 结果
- 生成：拼接查询 + 检索结果 → LLM 生成

优化方向：
- 分块策略：按段落/句子/语义分块
- 混合检索：语义 + BM25 + RRF 融合
- 重排序：Cross-Encoder 重排 Top-K 结果
- 查询改写：多查询、子问题分解
```

### 2. 请解释 LoRA（Low-Rank Adaptation）的原理，以及为什么它能减少显存占用。

**参考答案**：

```
LoRA 原理：
1. 冻结预训练模型的所有参数
2. 在每层的权重矩阵旁边添加低秩分解矩阵：W' = W + BA
   - W: 原始权重（d × d，冻结）
   - B: 低秩矩阵（d × r，可训练）
   - A: 低秩矩阵（r × d，可训练）
   - r: 秩（远小于 d，通常 4-64）

为什么减少显存：
1. 参数量：原模型 d² 参数 → LoRA 2×d×r 参数
   - 例：d=4096, r=16 → 参数量减少 128 倍
2. 梯度：只计算 BA 的梯度，不计算 W 的梯度
3. 优化器状态：只维护 BA 的优化器状态

效果：
- 显存占用减少 60-80%
- 训练速度提升 2-4 倍
- 性能接近全量微调（差距 < 1%）

合并：
- 训练完成后，可以将 BA 合并回 W：W' = W + BA
- 推理时无额外开销
```

### 3. 请解释 Multi-Head Attention 的作用，以及为什么需要多个 Head。

**参考答案**：

```
Single-Head Attention：
- Q, K, V 通过线性变换后计算注意力
- 每个 token 只能关注一种模式的关系

Multi-Head Attention：
- 将 Q, K, V 拆分为 h 个 head
- 每个 head 独立计算注意力
- 最后拼接所有 head 的输出

为什么需要多个 Head：
1. 不同的 head 可以学习不同的关系模式：
   - Head 1: 语法关系（主谓宾）
   - Head 2: 语义关系（同义词、反义词）
   - Head 3: 位置关系（相邻 token）
   - Head 4: 长距离依赖

2. 增加模型表达能力：
   - 单个 head 的表达能力有限
   - 多个 head 可以组合出更复杂的模式

3. 并行计算：
   - 每个 head 独立计算，可以并行
   - 不增加计算复杂度（总参数量不变）

数学表达：
head_i = Attention(QW_i^Q, KW_i^K, VW_i^V)
MultiHead(Q, K, V) = Concat(head_1, ..., head_h)W^O
```

### 4. 请解释 KV Cache 的原理，以及它如何加速 Transformer 推理。

**参考答案**：

```
问题：
- 自回归生成时，每个 token 需要计算与所有之前 token 的注意力
- 重复计算 K 和 V 是浪费的

KV Cache 原理：
1. 第一次计算时，缓存所有 token 的 K 和 V
2. 生成新 token 时，只计算新 token 的 Q, K, V
3. 用新 token 的 Q 与缓存的 K 计算注意力
4. 将新 token 的 K, V 追加到缓存

加速效果：
- 无 KV Cache：O(n²) 每步
- 有 KV Cache：O(n) 每步（n 是当前序列长度）

显存开销：
- 每层需要缓存 2 × seq_len × hidden_size × 2（FP16）
- 例：seq_len=2048, hidden=4096, 32层 → 约 1GB

优化：
- Multi-Query Attention（MQA）：所有 head 共享 K, V
- Grouped-Query Attention（GQA）：head 分组共享 K, V
- 量化：FP16 → INT8/INT4
```

```python
from collections import Counter, defaultdict

def bpe_tokenize(corpus: list[str], num_merges: int = 10) -> list[str]:
    """BPE 分词算法"""
    # 1. 初始化：将每个字符作为一个 token
    vocab = set()
    for word in corpus:
        for char in word:
            vocab.add(char)

    # 2. 统计字符对频率
    def get_pair_freqs(corpus):
        pair_freqs = Counter()
        for word in corpus:
            chars = list(word)
            for i in range(len(chars) - 1):
                pair = (chars[i], chars[i + 1])
                pair_freqs[pair] += 1
        return pair_freqs

    # 3. 合并最高频的字符对
    for _ in range(num_merges):
        pair_freqs = get_pair_freqs(corpus)
        if not pair_freqs:
            break

        # 找到最高频的对
        best_pair = max(pair_freqs, key=pair_freqs.get)
        new_token = best_pair[0] + best_pair[1]
        vocab.add(new_token)

        # 在语料库中合并这个对
        new_corpus = []
        for word in corpus:
            new_word = word.replace(best_pair[0] + best_pair[1], new_token)
            new_corpus.append(new_word)
        corpus = new_corpus

    return sorted(vocab)
```

```python
import numpy as np

def scaled_dot_product_attention(
    Q: np.ndarray,
    K: np.ndarray,
    V: np.ndarray,
    mask: np.ndarray = None
) -> tuple[np.ndarray, np.ndarray]:
    """Scaled Dot-Product Attention"""
    d_k = Q.shape[-1]

    # 1. 计算注意力分数
    scores = np.matmul(Q, K.T) / np.sqrt(d_k)

    # 2. 应用掩码（如果提供）
    if mask is not None:
        scores = np.where(mask, scores, -1e9)

    # 3. Softmax 归一化
    attention_weights = np.exp(scores) / np.sum(np.exp(scores), axis=-1, keepdims=True)

    # 4. 加权求和
    output = np.matmul(attention_weights, V)

    return output, attention_weights
```

```python
from typing import Callable, Dict, Any
import json

class Agent:
    def __init__(self, tools: Dict[str, Callable], llm_call: Callable):
        self.tools = tools
        self.llm_call = llm_call

    def run(self, query: str, max_iterations: int = 5) -> str:
        """运行 Agent 循环"""
        tools_desc = "\n".join([
            f"- {name}: {func.__doc__ or 'No description'}"
            for name, func in self.tools.items()
        ])

        history = [f"User: {query}"]

        for i in range(max_iterations):
            # 1. 构建 prompt
            prompt = f"""你是一个 AI Agent。可用工具：
{tools_desc}

格式：
Thought: <你的思考>
Action: <工具名>
Action Input: <输入>

或：
Thought: <你的思考>
Final Answer: <最终答案>

对话历史：
{chr(10).join(history)}"""

            # 2. 调用 LLM
            response = self.llm_call(prompt)
            history.append(f"Assistant: {response}")

            # 3. 解析响应
            if "Final Answer:" in response:
                answer = response.split("Final Answer:")[-1].strip()
                return answer

            # 4. 提取 Action
            action = None
            action_input = None
            for line in response.split("\n"):
                if line.startswith("Action:"):
                    action = line.split(":", 1)[1].strip()
                elif line.startswith("Action Input:"):
                    action_input = line.split(":", 1)[1].strip()

            if action and action in self.tools:
                try:
                    result = self.tools[action](action_input)
                except Exception as e:
                    result = f"Error: {e}"
                history.append(f"Observation: {result}")
            else:
                history.append(f"Observation: Unknown tool '{action}'")

        return "达到最大迭代次数"
```

```
RAG = Retrieval-Augmented Generation

检索阶段：
1. 索引构建：

   - HyDE（Hypothetical Document Embeddings）

3. 检索策略：
   - 语义检索：余弦相似度
   - 关键词检索：BM25
   - 混合检索：RRF 融合
   - 多路召回：不同检索器的结果合并

增强阶段：
1. 重排序：
   - Cross-Encoder 重排 Top-K 结果
   - 去重、过滤低质量结果

2. 上下文压缩：
   - LongLLMLingua：压缩检索到的文档
   - Extractive Compression：提取关键句子

3. 上下文拼接：
   - 检索结果 + 用户查询 → Prompt
   - 注意 Token 限制

生成阶段：
1. LLM 选择：
   - 小模型：快速、低成本
   - 大模型：高质量、高成本

2. 提示工程：
   - 指令明确：基于以下文档回答问题
   - 引用要求：标注来源
   - 否定语料：如果文档中没有信息，说"不知道"

3. 后处理：
   - 事实检查：验证生成内容是否与检索文档一致
   - 引用验证：检查引用的文档是否存在

常见优化：
1. 检索优化：
   - 分块策略：滑动窗口、语义分块
   - Embedding 微调：领域适配
   - 元数据过滤：按时间、来源、类型过滤

2. 生成优化：
   - Self-RAG：模型自己决定是否需要检索
   - CRAG（Corrective RAG）：检索后验证质量
   - Adaptive RAG：根据查询复杂度选择策略

3. 评估：
   - 检索质量：Recall@K、MRR、NDCG
   - 生成质量：Faithfulness、Relevance、Correctness
   - 端到端：EM（Exact Match）、F1
```
