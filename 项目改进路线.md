## 一、代码质量改进（必做）

### 1.1 合并两个沙箱实现

**现状**：存在两个独立的沙箱实现
- `services/code_sandbox.py`：静态扫描 + resource limits，被 `experiment_runner.py` 使用
- `core/sandbox.py`：namespace 隔离 + import hook，被 `base.py` / `data_agent.py` / `solver_agent.py` 使用

**问题**：
- 安全策略不一致（一个有 namespace，一个没有）
- API 接口不同，Agent 需要适配两种调用方式
- 维护成本翻倍，修 bug 要改两处

**方案**：
```python
# 统一为一个 CodeSandbox，支持配置化
class CodeSandbox:
    def __init__(self, config: SandboxConfig):
        self.use_namespace = config.enable_namespace  # 默认 True
        self.use_import_hook = config.enable_import_hook
        self.resource_limits = config.resource_limits
        self.static_scan = config.enable_static_scan
    
    async def execute(self, code: str, timeout: int = 120) -> SandboxResult:
        # 统一执行入口
        if self.use_namespace:
            return await self._execute_with_namespace(code, timeout)
        else:
            return await self._execute_with_subprocess(code, timeout)
```

**优先级**：P0（影响安全性和可维护性）
**面试价值**：解释"为什么合并"——单一职责原则 + 一致性

### 1.2 Token 估算精度提升

**现状**：`context_compressor.py` 用 `CHARS_PER_TOKEN = 3`（1 token / 3 字符）

**问题**：中英文混合时误差可达 2x。中文 1 个字 ≈ 2-3 tokens，英文 1 个单词 ≈ 1-1.5 tokens。粗略估算可能导致：
- 不必要的压缩（实际没超限）
- 上下文溢出（实际已超限但没检测到）

**方案**：
```python
# 方案 A：使用 tiktoken（OpenAI 的分词器）
import tiktoken
ENCODING = tiktoken.get_encoding("cl100k_base")

def count_tokens(text: str) -> int:
    return len(ENCODING.encode(text))

# 方案 B：轻量估算（不引入依赖）
def estimate_tokens(text: str) -> int:
    # 英文：按空格分词，中文：按字分词
    english_words = len(re.findall(r'[a-zA-Z]+', text))
    chinese_chars = len(re.findall(r'[一-鿿]', text))
    other_chars = len(text) - english_words * 5 - chinese_chars  # 粗略
    return english_words + chinese_chars * 2 + other_chars // 4
```

**优先级**：P1（影响上下文管理可靠性）
**面试价值**：展示对 tokenization 的理解

## 二、功能完善（高价值）

### 2.1 向量检索替代关键词匹配

**现状**：`LessonsMemory.retrieve_relevant()` 用关键词匹配 + 滑动窗口

**问题**：
- "如何处理 LLM 幻觉" 和 "减轻模型编造数据" 语义相似但关键词不同，无法匹配
- 中文分词效果差，滑动窗口提取的关键词质量低

**方案**：✅ 已修复（EpisodicMemory 现在支持 LLM 智能摘要）
```python
# 使用 sentence-transformers 做语义检索
from sentence_transformers import SentenceTransformer
import numpy as np

class VectorLessonsMemory:
    def __init__(self):
        self.model = SentenceTransformer('all-MiniLM-L6-v2')
        self.lessons = []  # 原始经验
        self.embeddings = None  # 缓存的嵌入向量
    
    def add_lesson(self, lesson: dict):
        self.lessons.append(lesson)
        self.embeddings = None  # 使缓存失效
    
    def _build_index(self):
        texts = [l['description'] for l in self.lessons]
        self.embeddings = self.model.encode(texts)
    
    def retrieve(self, query: str, top_k: int = 5) -> list:
        if not self.lessons:
            return []
        if self.embeddings is None:
            self._build_index()
        
        query_embedding = self.model.encode([query])
        similarities = np.dot(self.embeddings, query_embedding.T).flatten()
        top_indices = np.argsort(similarities)[-top_k:][::-1]
        
        return [self.lessons[i] for i in top_indices]
```

**优先级**：P1（大幅提升记忆系统质量）
**面试价值**：核心创新点，能讲 10 分钟

### 2.2 SSE 实时进度推送完善

**现状**：`routers/tasks.py` 有 `StreamingResponse` 导入，但实际推送逻辑不完整

**问题**：用户提交任务后只能轮询状态，无法实时看到 Agent 执行进度

**方案**：
```python
# 基于 asyncio.Queue 的事件推送
class TaskEventBus:
    def __init__(self):
        self.queues: Dict[str, asyncio.Queue] = {}
    
    def subscribe(self, task_id: str) -> asyncio.Queue:
        queue = asyncio.Queue()
        self.queues[task_id] = queue
        return queue
    
    async def publish(self, task_id: str, event: dict):
        if task_id in self.queues:
            await self.queues[task_id].put(event)
    
    def unsubscribe(self, task_id: str):
        self.queues.pop(task_id, None)

# SSE 端点
@app.get("/tasks/{task_id}/stream")
async def stream_events(task_id: str):
    bus = get_event_bus()
    queue = bus.subscribe(task_id)
    
    async def event_generator():
        try:
            while True:
                event = await asyncio.wait_for(queue.get(), timeout=30)
                yield f"data: {json.dumps(event)}\n\n"
                if event.get('type') == 'completed':
                    break
        finally:
            bus.unsubscribe(task_id)
    
    return StreamingResponse(event_generator(), media_type="text/event-stream")
```

**优先级**：P1（用户体验核心）
**面试价值**：SSE vs WebSocket 的选择理由

class AsyncTaskManager:

    async def submit(self, task_id: str, coro):
        """提交异步任务"""
        task = asyncio.create_task(self._run_with_persistence(task_id, coro))

### 4.1 LLM 输出校验强化

**现状**：Anti-Fabrication 检测无来源数据，但覆盖不全面

**方案**：✅ 已修复（MCP fallback + 推理脚本 + NAS/Loss 评估器 + Episodic Memory）
```python
class OutputValidator:
    """LLM 输出综合校验"""
    
    def validate(self, agent_name: str, output: dict) -> ValidationResult:
        errors = []
        warnings = []
        
        # 1. 必填字段检查
        required_fields = self._get_required_fields(agent_name)
        for field in required_fields:
            if field not in output or not output[field]:
                errors.append(f"Missing required field: {field}")
        
        # 2. 类型检查
        for key, value in output.items():
            expected_type = self._get_expected_type(agent_name, key)
            if expected_type and not isinstance(value, expected_type):
                errors.append(f"Type mismatch: {key} should be {expected_type}")
        
        # 3. 数值范围检查
        if "accuracy" in output:
            if output["accuracy"] > 1.0 or output["accuracy"] < 0:
                errors.append(f"Invalid accuracy: {output['accuracy']}")
        
        # 4. JSON 格式检查（如果要求 JSON 输出）
        if output.get("_requires_json"):
            try:
                json.loads(output.get("content", "{}"))
            except json.JSONDecodeError:
                errors.append("Invalid JSON format in output")
        
        # 5. 长度检查
        content = output.get("content", "")
        if len(content) > 50000:
            warnings.append(f"Output too long: {len(content)} chars")
        
        return ValidationResult(errors=errors, warnings=warnings)
```

**优先级**：P1（防幻觉核心）
**面试价值**：Anti-Fabrication 设计的深化

### 4.2 API 限流

**现状**：无 LLM 调用频率限制，依赖熔断器的事后检测

**方案**：
```python
import time
from collections import defaultdict

class RateLimiter:
    """令牌桶限流器"""
    
    def __init__(self, rpm: int = 60, tpm: int = 100000):
        self.rpm = rpm  # 每分钟请求数限制
        self.tpm = tpm  # 每分钟 token 数限制
        self.request_timestamps = defaultdict(list)
        self.token_counts = defaultdict(int)
    
    async def acquire(self, provider: str, estimated_tokens: int = 1000):
        """获取执行许可"""
        now = time.time()
        window = 60  # 1 分钟窗口
        
        # 清理过期记录
        self.request_timestamps[provider] = [
            ts for ts in self.request_timestamps[provider] 
            if now - ts < window
        ]
        
        # 检查请求数限制
        if len(self.request_timestamps[provider]) >= self.rpm:
            wait_time = window - (now - self.request_timestamps[provider][0])
            await asyncio.sleep(wait_time)
        
        # 检查 token 数限制
        if self.token_counts[provider] + estimated_tokens > self.tpm:
            await asyncio.sleep(10)  # 等待 10 秒
        
        # 记录本次请求
        self.request_timestamps[provider].append(now)
        self.token_counts[provider] += estimated_tokens
```

**优先级**：P1（防止 API 配额耗尽）
**面试价值**：展示对生产环境的理解

### 5.1 LLM 调用缓存

**现状**：相同 prompt 每次都调用 LLM

**问题**：开发调试时重复调用浪费 token 和时间

**方案**：
```python
import hashlib
import json

class LLMCache:
    """LLM 调用结果缓存"""
    
    def __init__(self, cache_dir: str = "backend/data/llm_cache"):
        self.cache_dir = Path(cache_dir)
        self.cache_dir.mkdir(parents=True, exist_ok=True)
    
    def _make_key(self, messages: list, model: str) -> str:
        content = json.dumps({"messages": messages, "model": model}, sort_keys=True)
        return hashlib.sha256(content.encode()).hexdigest()
    
    async def get_or_call(self, client, messages, model, **kwargs):
        key = self._make_key(messages, model)
        cache_path = self.cache_dir / f"{key}.json"
        
        # 检查缓存
        if cache_path.exists():
            with open(cache_path) as f:
                return json.load(f)
        
        # 调用 LLM
        result = await client.chat(messages=messages, model=model, **kwargs)
        
        # 写入缓存
        with open(cache_path, 'w') as f:
            json.dump(result, f, ensure_ascii=False, indent=2)
        
        return result
```

**优先级**：P2（开发效率）
**面试价值**：展示缓存设计能力

## 八、面试中如何谈改进

### 8.1 如果面试官问"你的项目有什么不足？"

```
诚实回答当前剩余的不足：

1. "NAS 和 Loss Design 的评估器虽然是真实训练实现，
    但受限于单卡 8GB 显存，搜索深度有限（种群≤10，代数≤5）。
    改进方案是接入 GPU 集群扩大搜索空间。"

2. "端到端集成测试还没做——用 mock LLM 跑完整工作流的测试。
    目前测试主要覆盖基础设施和单个 Agent，多 Agent 协作的
    集成路径还没有系统性验证。"

3. "记忆检索用的是关键词匹配 + LLM 摘要，不是纯向量语义检索。
    改进方案是接入 sentence-transformers 做余弦相似度检索。"
```

### 8.2 如果面试官问"如果再给你一周时间，你会做什么？"

```
"我会做两件事：

1. 端到端测试：用 mock LLM 跑完整工作流，
   确保 15 个 Agent 的集成是可靠的。
   目前测试主要覆盖基础设施，Agent 执行路径几乎没有测试。

2. 向量检索升级：用 sentence-transformers 替代关键词匹配，
   让经验检索从'字面匹配'升级为'语义理解'。
   这能让系统真正'学到东西'。"
```

### 8.3 如果面试官问"你觉得这个项目最大的技术挑战是什么？"

```
"最大的挑战不是单个模块的实现，而是 15 个模块的集成。

举个例子：当 solver_agent 执行失败时，
需要同时处理：
- 错误分类（是代码 bug 还是依赖问题？）
- 重试策略（最多重试几次？间隔多久？）
- 熔断器状态（是否触发熔断？）
- 检查点保存（恢复时从哪里继续？）
- 上下文传递（错误信息如何注入下一次 LLM 调用？）

这些逻辑分散在 base.py、circuit_breaker.py、
task_persistence.py、context_compressor.py 等多个模块中，
要保证它们协同工作且不出 bug，是最难的部分。"
```
